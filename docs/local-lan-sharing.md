# 같은 LAN(공유기) 안에서의 접속·공유

이 문서는 **새 대화에서도** 아래 맥락을 잃지 않기 위한 **프로젝트 의도**를 고정해 둔다.

## 목표

- **같은 Wi-Fi / 같은 유선·무선 공유기(같은 LAN)** 에 붙은 기기끼리만 게임(또는 갤러리 페이지)에 접속하면 된다.
- 예: `http://192.168.0.43:3001/gallery.html` 처럼 **호스트 PC의 사설 IP + 포트 + 경로**를 상대에게 알려 주면, **같은 LAN 안**에서는 그 URL로 들어가 보면 된다.
- `localhost`로만 서버를 띄우는 것과 **다른 사람이 접속하는 주소**는 다르다. 상대 PC의 `localhost`는 **상대 PC 자신**을 가리킨다.

## 인터넷 전체 공개와의 구분

| 방식 | 누가 접속 가능한가 |
|------|-------------------|
| `http://127.0.0.1` / `http://localhost` | 그 PC에서만 (본인) |
| `http://192.168.x.x:포트/경로` (사설 IP) | **같은 LAN**에 있는 기기 (집/강의실·회의실 등) |
| Vercel·Netlify 등 **배포 URL** | 인터넷만 되면 어디서나 |

**이 프로젝트에서 “공유”의 1차 의도는 같은 LAN**이다. 수업 밖·원격에서 누구나 열어야 하면 **별도의 정적 배포 URL**이 필요하다(레포의 스펙·`Temp` 기획과 `README` 참고).

## 기술적으로 필요한 점(요약)

1. **개발/정적 서버는 LAN에서 들어올 수 있게 바인딩**  
   `127.0.0.1`에만 묶이면 타 기기에서 `192.168.x.x`로 접속할 수 없다. 도구에 따라 `0.0.0.0` 또는 `--host` 등으로 **모든 인터페이스**에서 듣는 설정이 필요하다.
2. **`192.168.0.43` 같은 주소는 바뀔 수 있음** (DHCP). 자주 쓰면 공유기에서 해당 기기 **고정 IP(예약)** 를 두는 편이 덜 헷갈린다.
3. **OS 방화벽**이 해당 **포트(예: 3001)** 인바운드를 막으면, 같은 Wi-Fi여도 접속이 안 될 수 있다. 필요 시 방화벽에서 허용한다.
4. **경로** (`/gallery.html` 등)는 실제로 서버가 그 파일(또는 라우트)을 제공하는 구조와 맞아야 한다.
5. **HTTPS가 필요한 브라우저 API**는 `http` + 사설 IP 조합에서 제한될 수 있다. (순수 정적/Canvas 위주·Dreamlo `fetch` 등은 대부분 LAN HTTP로도 실용적이다.)

## 백엔드(Dreamlo·Supabase 등)

- 랭킹/백엔드는 **이미 인터넷上的 서비스**라면, **프론트만 LAN IP로 열고** API는 그대로 호출하는 식으로 동작하는 경우가 많다.
- **CORS / 키 노출**은 각 서비스·호스트 설정에 맞게 점검한다.

## MacBook 트랙패드(시야 좌·우)

- **증상**: 가로로 돌릴 때 멈추듯 하거나, 탭 **뒤로/앞으로**로 가는 느낌이면 브라우저 **히스토리 제스처**·가로 `wheel` 경로가 겹친 경우가 많다. `web/index.html`의 `overscroll-behavior`와 `web/src/game/input.js`의 `wheel` `deltaX` 처리로 완화한다.
- **확인**: **마우스**로는 재현되는지 비교해 본다. **시스템 설정 → 트랙패드**에서 “**페이지 간 쓸어넘기**”(또는 “스와이프 쓸어넘기”, 브라우저마다 OS 문구는 다름)을 끄고, **Chrome vs Safari**에서도 비교해 본다.
- **포인터 락**이 풀리면 플레이 중 **게임 캔버스를 다시 클릭**해 잠금을 복구할 수 있다.
